About the Project

This website visualizes the results of a project of mine, where I automatically download (scrape) and analyze data from the handball.ch matchcenter. I use Python (Selenium Webdriver, Pandas, Matplotlib) to collect the data and visualize it statically, as well as JavaScript (D3.js) to create the interactive visualizations on this website.

In the highest leagues of Swiss Handball, games (and player stats) are recorded using a feature called 'Liveticker'. Since I coach a team in such a league, I figured I would use this data, visualize it interactively and maybe gain some more insights into the performance of my team. Sadly, I could not find an API to get the data directly, so I had to resort to scraping it. Along the way I implemented a mechanism for data collection that lets me analyze additional teams with minimal effort, so therefore I ended up analyzing more than just my team's performance.

What I am proud of

This webpage was designed to need as little maintenance as possilbe. The Dropdown menus where users can select teams, seasons and statistics, for example, are generated automatically. That is, if I push new data to the repository, these dropdowns are instantly up to date. No need to change a thing for this website.

Of course, I am equally as proud of writing a program that can gather and visualize data automatically from a website!

Limitations, or: What I am not so proud of

The page is currently only half mobile-ready. I will try to find a way to make it better for mobile usage - however this may prove to be rather difficult.

Furthermore, I would like to make it so the runner is executed automatically every Saturday and Sunday, so the whole workflow is fully automatic.

About the Data

Aquisition

Data is aquired using a selenium webkit based webscraper, which is gathering all kinds of statistics from the handball.ch liveticker feature. All that needs to change in order to get statistics for another team is the options.py file in root of the handballStats repository. Everything else is handled dynamically. What I can see for the future is having a shell script run a smaller options.py on a Raspberry Pi 3 with only the current seasons of teams I'm interested in - half an hour to an hour after any of these teams completes a game.

Processing

Data gathered this way is processed using python (pandas, matplotlib) and visualized a first time in the output_png folder in the repository.

Visualizations

As mentioned above, the data is visualized a first time using python and matplotlib. The much more interesting, interactive visualizations on this website are generated client-side with D3.js.
